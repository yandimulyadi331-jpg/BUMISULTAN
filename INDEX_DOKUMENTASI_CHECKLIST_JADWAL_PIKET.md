## ğŸ“‹ KEY CONCEPTS GLOSSARY

### **Periode Key**
```
Format: "harian_{date}_{kodeJamKerja}"
Contoh: "harian_2026-01-22_NONS"

Tujuan: Unique identifier per shift, memastikan tidak ada mix-up
Benefit: Easy cleanup, accurate audit trail
```

### **Kode Jam Kerja**
```
NONS = NON SHIFT (08:00 - 17:00)
SFT1 = SHIFT 1 (08:00 - 20:00)
SFT2 = SHIFT 2 (20:00 - 08:00) â† Cross-midnight
```

### **Validation Layers**
```
Layer 1: User is employee?
Layer 2: Has presensi today?
Layer 3: Current time in work hours?
Layer 4: Periode status is ACTIVE?
Layer 5: Checklist is for user's shift?
```

### **Status Periode**
```
ACTIVE = Dapat dikerjakan
CLOSED = Tidak dapat dikerjakan (auto-lock setelah jam kerja)
```

### **Force Pulang Outcomes**
```
- Semua selesai â†’ Pulang Lebih Awal - Valid (+bonus points)
- Belum selesai + dalam jam kerja â†’ Pulang dengan catatan (-penalty)
- Diluar jam kerja â†’ Periode closed (tidak bisa pulang normal)
```
